  • Minimal version (Includes OA), this might be sufficient for you if you only want the wide area database related functionalities.

Or, you can go to our CVS to dig out the version you want.


Instructions to download from CVS:


  • What hardware/software do I need to install IrisNet?
    To build IrisNet, you need a computer with C/C++ compiler. We develop IrisNet on Linux, so our current version installs smooth there. In future we will try to port the code to other platforms.

  • How can I get the IrisNet software?
    You can download IrisNet code build it on your computer. IrisNet code is available both by HTTP and by anonymous CVS. The whole source code tree can be browsed here.
     
    There are two ways to buils IrisNet: from all the pieces or all at once. IrisNet is composed of multiple independent modules, and those modules can be downloaded independently.

  • What are the different IrisNet modules?
    Currently, IrisNet has four main modules: OA (Organizing Agent module), SA (Sensing Agent module), Logger (Event logging module, used for debugging), and Applications (different applications we have implemenetd so far on IrisNet). The OA module is written in Java, and the rest of the modules are written in C/C++.

  • Does IrisNet need other third party packages?
    Yes. IrisNet leverages the following existing off-the-shelf packages.
                    
Package Module that requires the package
Apache Xindice (XML database engine) OA
Intel OpenCV (computer vision library) Applications
NAM (Network Animator) Logger
CamStream (webcam viewer) SA

             
Since Xindice provides Java APIs for JDK-1.3, IrisNet requires that specific version of JDK too.


  • How can I get the code via anonymous cvs?
    First, make sure that you have the cvs client installed in your machine. (do "whereis cvs")
    Then, set the environment variable CVSROOT as ":pserver:anonymous@iris.wv.cc.cmu.edu:/home/iris/cvs". Finally, do "cvs login" with the password "anonymous". You are all set to check out IrisNet code from the CVS repository.
     
    Now, you are to decide which module you want to checkout. The whole IrisNet package (including all the third party packages it depends on) is around 300MB in size, so you may like to checkout only the modules you are interested in. Depending on what module you want to checkout, use the CVS command "cvs co -r V200 <module_name>", where V200 is the current stable release of IrisNet code and module_name is any of the following:
             
Module name Checked out code
IRISNET-all All the modules (OA, SA, Logger, Applications) and all the third party packages they need
OA-all The OA modules and all the third party packages it needs (Xindice, JDK1.3 etc.)
OA The OA module (The 3rd party packages are not downloaded. Make sure that they are installed in your machine)
SA-all The SA modules and all the third party packages it needs. (CamStream etc.)
SA The SA module (The 3rd party packages are not downloaded. Make sure that they are installed in your machine)
Logger-all The Logger modules and all the third party packages it needs (NAM etc.)
Logger The Logger module (The 3rd party packages are not downloaded. Make sure that they are installed in your machine)
Applications-all The Application modules and all the third party packages it needs (Intel OpenCV etc.)
Applications The application module. (The 3rd party packages are not downloaded. Make sure that they are installed in your machine)

             
We recommend you to check out the "-all" modules, which incorporate all the necessary packages. In case you do not checkout the 3rd party packages, the install script should find them automatically. If not, please let us know so that we can fix that in the install scripts.
 
You can check out the 3rd party packages even if you already have those packages installed in your machine. IrisNet's install script installs everything under the IrisNet directory, so existing packages are not supposed to conflict with the packages that come with IrisNet.


  • Can I commit my changes to the CVS repository?
    No. As an anonymous user, you do not have write access to the reposritory. If you want to contribute to our code, contact     us.

  • O.K. I have checked out the code. How can I install the code (and 3rd party packages) now?
    Within the IrisNet directory, type "./install", and all the modules you checked out should be installed.

  • I did run "./install", but the installation failed. So now..?
    Read the FAQ page. If you still have problem in installing the IrisNet code, please send a mail to irisdev@intel-iris.net
  • How can I uninstall the modules? (and 3rd party packages?)
    Within the IrisNet Directory, type "./uninstall".

  • Where should I send the bug reports/feature requests/suggestions/codes?
    Mail them to us.
